# Contributing

Thanks for improving this project.

## Contribution Principles

- Keep the template simple enough for small businesses.
- Do not add features that require sensitive data by default.
- Prefer staff-approved workflows over fully automatic customer messaging.
- Add validation and logging when adding automation.
- Update docs when changing sheet names, headers, or Apps Script behavior.

## Pull Request Checklist

- [ ] No real personal information is included.
- [ ] Example data is fake.
- [ ] Apps Script syntax was checked.
- [ ] Security or privacy impact was considered.
- [ ] Documentation was updated.
- [ ] QA test cases were updated if behavior changed.

## Development Notes

The Apps Script file is in:

```text
apps-script/reservation_reminder_demo.gs
```

When changing sheet names or headers, update:

- `README.md`
- `docs/setup/apps_script_setup.md`
- `docs/qa_test_cases.md`
- `docs/security/security_checklist.md`
