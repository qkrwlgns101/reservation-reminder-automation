# 예약 자동화 패키지 다음 단계 실행 계획

## 결론

다음 단계는 `실제 Google Sheets 데모 구축 → 파일럿 고객 10곳 접촉 → 1곳 무료/저가 적용 → 유료 전환`입니다.

지금은 기능을 더 만드는 것보다, 고객이 돈을 낼지 검증하는 단계입니다.

## 1단계: 실제 데모 환경 만들기

목표: 고객에게 화면 공유로 보여줄 수 있는 실제 Google Sheets + Google Form 데모를 만든다.

작업:

1. Google Drive에 새 폴더 생성
   - 폴더명: `예약 누락 방지 자동화 데모`

2. Google Sheets 생성
   - 파일명: `예약 누락 방지 자동화 데모`
   - 기존 `reservation_reminder_template.xlsx` 구조를 Google Sheets로 옮긴다.

3. Google Form 생성
   - 제목: `예약 접수 데모 폼`
   - `google_form_blueprint.md` 기준으로 질문 구성
   - 응답을 Google Sheets에 연결

4. Apps Script 설치
   - `reservation_reminder_demo.gs` 붙여넣기
   - 메뉴 생성 확인
   - `사전 점검 실행`
   - `보안 점검 실행`
   - `백업 스냅샷 생성`
   - `마스킹 뷰 생성`

5. 샘플 데이터 입력
   - 오늘 예약 3건
   - 내일 예약 3건
   - 미방문 1건
   - 수신동의 FALSE 1건
   - 잘못된 데이터 1건을 넣어 점검 기능 시연

완료 기준:

- Form 제출 시 예약 데이터가 시트에 반영된다.
- 발송 대기함에 안내 문구가 생성된다.
- 보안 점검 결과가 `Security_Audit`에 남는다.
- 마스킹 뷰에서 이름/연락처가 가려진다.

## 2단계: 첫 타깃 업종 하나만 고르기

초기에는 범용으로 팔려고 하지 말고, 한 업종에 꽂아서 팔아야 합니다.

추천 순서:

1. 치과
2. 피부과
3. 학원
4. 미용실/네일샵
5. PT/필라테스

첫 파일럿은 `치과` 또는 `피부과`를 추천합니다.

이유:

- 예약제 운영이 명확하다.
- 노쇼와 전날 안내 문제가 실제로 있다.
- 직원이 수동으로 문자/카톡 안내를 많이 보낸다.
- 보안 설계를 강조하면 신뢰를 얻기 쉽다.

## 3단계: 무료 진단 제안 10곳 보내기

목표: 판매보다 먼저 상담 약속을 잡는다.

보낼 대상:

- 동네 치과 5곳
- 피부과/의원 5곳

보낼 메시지:

```text
안녕하세요. 예약제 병원용으로 예약 누락과 전날 안내 업무를 줄이는 Google Sheets 기반 자동화 시스템을 만들고 있습니다.

예약이 들어오면 자동으로 정리되고,
전날 안내/당일 안내/미방문 후속 문구가 자동 생성되는 방식입니다.

진료 판단이나 환자 상담을 AI가 하는 것이 아니라,
직원이 반복해서 처리하는 예약 안내 업무를 줄이는 도구입니다.

개인정보 보호를 위해 최소 정보만 수집하고,
마스킹 뷰, 보안 점검, 백업 기능까지 포함해 설계했습니다.

현재 3곳만 무료로 예약 관리 방식 진단을 해드리고 있는데,
15분 정도 현재 예약 흐름을 듣고 개선 가능한 부분 3가지만 정리해드려도 괜찮을까요?
```

## 4단계: 상담 때 보여줄 순서

상담은 15분 이내로 끝냅니다.

순서:

1. 현재 예약 관리 방식 질문
2. Google Form 예약 접수 화면 보여주기
3. 예약 목록 자동 정리 화면 보여주기
4. 발송 대기함 보여주기
5. 보안 점검/마스킹 뷰 보여주기
6. 주간 리포트 보여주기
7. 파일럿 제안

마무리 멘트:

```text
처음부터 큰 시스템을 도입하는 방식이 아니라,
현재 예약 안내 업무 중 가장 반복적인 부분만 Google Sheets로 정리해보는 방식입니다.
3~7일 정도면 파일럿 적용이 가능하고, 효과가 없으면 확장하지 않으셔도 됩니다.
```

## 5단계: 첫 파일럿 제안

첫 고객은 큰돈보다 사례 확보가 중요합니다.

추천 제안:

- 구축비: 30만 원
- 유지보수: 첫 달 무료
- 조건: 후기 또는 익명 사례 사용 허락
- 기간: 7일 구축 + 2주 운영

파일럿 범위:

- 예약 접수 폼
- 예약 관리 시트
- 발송 대기함
- 미방문 후속 문구
- 주간 리포트
- 보안 점검
- 마스킹 뷰

## 6단계: 유료 전환 기준

파일럿 2주 후 아래 질문으로 유료 전환을 시도합니다.

질문:

1. 전날 안내 업무가 줄었나요?
2. 예약 누락 확인이 쉬워졌나요?
3. 미방문 고객 후속 연락이 더 잘 보이나요?
4. 직원이 계속 사용할 수 있겠나요?
5. 다음 달에도 유지보수가 필요할까요?

유료 전환 제안:

- 월 관리비 10만~20만 원
- 문구 수정
- 오류 점검
- 월간 리포트
- 직원 질문 대응

## 7단계: 이번 주 할 일

오늘:

- Google Sheets 실제 데모 만들기
- Google Form 만들기
- Apps Script 붙여넣기

내일:

- 샘플 데이터 입력
- 보안 점검/오류 테스트
- 화면 공유용 데모 흐름 연습

3일차:

- 치과/피부과 10곳 리스트 작성
- 무료 진단 메시지 발송

4~7일차:

- 답장 온 곳 상담
- 1곳 파일럿 제안
- 피드백 기록
