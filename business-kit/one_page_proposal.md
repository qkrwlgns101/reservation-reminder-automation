# 예약 누락 방지 자동화 패키지 제안서

## 제안 요약

예약제 사업장은 매일 반복해서 예약 확인, 전날 안내, 당일 안내, 취소/미방문 후속 연락을 처리합니다. 이 업무가 직원 기억과 수기 관리에 의존하면 예약 누락, 안내 누락, 노쇼, 고객 응대 품질 저하가 생깁니다.

`예약 누락 방지 자동화 패키지`는 Google Forms와 Google Sheets를 이용해 예약 정보를 자동으로 정리하고, 오늘 보내야 할 안내 문구와 주간 운영 리포트를 자동으로 보여주는 시스템입니다.

## 이런 사업장에 적합합니다

- 예약 문의가 전화, 카톡, 네이버, 홈페이지 등 여러 곳에서 들어온다.
- 전날/당일 안내 문자를 직원이 매번 직접 작성한다.
- 예약 취소나 미방문 고객 후속 연락이 자주 누락된다.
- 원장/대표가 주간 예약 현황을 한눈에 보고 싶어 한다.
- 별도 복잡한 프로그램보다 익숙한 Google Sheets로 관리하고 싶다.

## 제공 기능

| 기능 | 설명 |
|---|---|
| 예약 접수 폼 | 고객명, 연락처, 예약일, 예약시간, 예약 항목을 접수 |
| 예약 관리 시트 | 예약 상태, 담당자, 유입 경로, 메모를 한 화면에서 관리 |
| 자동 안내 문구 | 예약확정, 전날안내, 당일안내, 미방문안내, 리뷰요청 문구 생성 |
| 발송 대기함 | 오늘 보내야 할 메시지를 자동으로 정리 |
| 미방문 관리 | 미방문 고객에게 보낼 재예약 안내 문구 생성 |
| 주간 리포트 | 예약 수, 완료 수, 취소 수, 미방문 수, 미방문율 집계 |
| 업종별 템플릿 | 병원, 학원, 미용실, 상담센터 등 업종별 문구 적용 |

## 운영 방식

1. 직원 또는 고객이 예약 폼을 작성합니다.
2. 예약 정보가 Google Sheets에 자동 저장됩니다.
3. 시스템이 예약 안내 문구를 자동 생성합니다.
4. 직원은 발송 대기함에서 문구를 확인한 뒤 문자/카톡으로 발송합니다.
5. 방문 후 상태를 완료, 취소, 미방문으로 바꿉니다.
6. 대표자는 대시보드와 주간 리포트로 운영 현황을 확인합니다.

## 안전한 시작 방식

초기 버전은 자동 문자 발송이 아니라 `직원 확인 후 발송` 방식입니다. 고객에게 바로 메시지가 나가지 않기 때문에 문구 오류, 개인정보 이슈, 업종별 표현 문제를 줄일 수 있습니다.

또한 예약 관리에 필요한 최소 정보만 수집합니다.

- 이름
- 연락처
- 예약일
- 예약시간
- 예약 항목
- 수신 동의 여부

진료 내용, 질병명, 검사 결과, 민감한 상담 내용은 저장하지 않는 것을 기본으로 합니다.

## 보안 설계

개인정보가 포함되는 원본 시트는 필요한 직원에게만 공유하고, 외부 공유가 필요한 경우에는 이름과 연락처가 가려진 `Masked_View`를 사용합니다.

기본 보안 기능:

- 링크 공유 위험 점검
- 민감정보 의심 입력 점검
- 보관기간 초과 데이터 표시
- 수정 전 백업 스냅샷 생성
- 실행 이력과 오류 로그 기록
- 주요 시트와 개인정보 범위 보호 경고
- 고객명/연락처 마스킹 뷰 생성

## 예상 효과

- 반복 안내 연락 시간 감소
- 전날/당일 안내 누락 감소
- 미방문 고객 후속 연락 체계화
- 직원별 예약 관리 방식 통일
- 주간 예약 현황을 빠르게 확인

## 상담 제안

현재 예약 관리 방식을 15분만 확인하면, 어떤 부분을 자동화할 수 있는지 바로 진단해드릴 수 있습니다.

진단 후에는 실제 샘플 화면으로 `예약 접수 → 발송 대기함 → 주간 리포트` 흐름을 보여드리겠습니다.
