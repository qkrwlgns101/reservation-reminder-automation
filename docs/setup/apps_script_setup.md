# Apps Script 설치 안내

## 목적

`reservation_reminder_demo.gs`는 Google Sheets에서 아래 자동화를 시연하기 위한 샘플 코드입니다.

- 메뉴 추가
- 사전 점검 실행
- 보안 점검 실행
- 보호 설정 적용
- 마스킹 뷰 생성
- 백업 스냅샷 생성
- 발송 대기함 자동 생성
- 메시지 템플릿 변수 치환
- 주간 리포트 갱신
- 실행 이력 기록
- 매일 오전 자동 실행 트리거 설치

## 설치 순서

1. Google Sheets에서 예약관리 템플릿을 엽니다.
2. 상단 메뉴에서 `확장 프로그램 > Apps Script`를 엽니다.
3. 기본 코드 파일 내용을 삭제합니다.
4. `reservation_reminder_demo.gs` 내용을 붙여 넣습니다.
5. 저장합니다.
6. Google Sheets로 돌아와 새로고침합니다.
7. 상단 메뉴에 `예약 자동화`가 생기면 설치 성공입니다.

## 최초 실행

상단 메뉴에서 아래 순서로 실행합니다.

1. `예약 자동화 > 발송 대기함 갱신`
2. `예약 자동화 > 주간 리포트 갱신`

권한 승인 창이 뜨면 Google 계정 권한을 승인해야 합니다.

## 매일 자동 갱신 설정

상단 메뉴에서:

```text
예약 자동화 > 매일 오전 트리거 설치
```

을 클릭하면 매일 오전 8시~9시 사이에 발송 대기함과 주간 리포트가 자동 갱신됩니다.

## 납품 전 점검 순서

고객에게 넘기기 전 아래 순서로 확인합니다.

1. `예약 자동화 > 백업 스냅샷 생성`
2. `예약 자동화 > 사전 점검 실행`
3. `예약 자동화 > 보안 점검 실행`
4. `Validation_Errors` 시트에 BLOCKER 오류가 없는지 확인
5. `Security_Audit` 시트에 HIGH/CRITICAL 위험이 없는지 확인
6. `예약 자동화 > 보호 설정 적용`
7. `예약 자동화 > 마스킹 뷰 생성`
8. `예약 자동화 > 발송 대기함 갱신`
9. `예약 자동화 > 주간 리포트 갱신`
10. `Audit_Log` 시트에 SUCCESS 기록이 남는지 확인

## 오류가 생겼을 때

- `Validation_Errors` 시트에서 오류 위치와 원인을 확인합니다.
- `BLOCKER` 오류는 먼저 수정한 뒤 자동화를 다시 실행합니다.
- `WARN` 오류는 운영자가 확인 후 진행할 수 있습니다.
- 수식이나 데이터가 크게 꼬였으면 백업 스냅샷 시트와 비교해 복구합니다.

## 시트 이름

코드는 아래 시트 이름을 기준으로 작동합니다.

- `Settings`
- `Appointments`
- `Message_Templates`
- `Send_Queue`
- `Weekly_Report`
- `Validation_Errors`
- `Audit_Log`
- `Security_Audit`
- `Masked_View`

## 보안 메뉴 설명

| 메뉴 | 설명 |
|---|---|
| 사전 점검 실행 | 필수 시트, 필수 헤더, 중복 예약 ID, 잘못된 상태값을 확인합니다. |
| 보안 점검 실행 | 파일 공유 상태, 민감정보 의심 입력, 연락처 형식, 보관기간 초과 데이터를 확인합니다. |
| 보호 설정 적용 | 대시보드, 로그, 리포트, 주요 개인정보 범위에 수정 경고를 설정합니다. |
| 마스킹 뷰 생성 | 고객명과 연락처를 가린 외부 공유용 시트를 생성합니다. |

## 보안 운영 주의사항

- 원본 시트는 링크 공유하지 않습니다.
- 외부 공유가 필요하면 원본 대신 `Masked_View`를 사용합니다.
- 고객 요청사항에는 진료 내용, 질병명, 검사결과, 주민등록번호, 결제정보를 입력하지 않습니다.
- 납품 후 외부 작업자 권한을 회수합니다.

시트 이름을 바꾸면 코드도 함께 수정해야 합니다.

## 주의사항

- 이 코드는 문자/카카오를 실제로 자동 발송하지 않습니다.
- 발송할 문구를 `Send_Queue`에 생성하는 데모용 코드입니다.
- 고객에게 메시지를 보내기 전 직원이 반드시 문구를 확인해야 합니다.
- 민감정보, 진료정보, 질병명, 검사결과는 시트에 저장하지 않는 운영 방식을 권장합니다.
