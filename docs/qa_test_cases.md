# 오류 테스트 케이스

## 테스트 목적

고객 납품 전 아래 상황을 일부러 만들어 보고, 시스템이 조용히 망가지지 않고 오류를 감지하거나 복구 가능한지 확인합니다.

## 테스트 결과 표기

| 상태 | 의미 |
|---|---|
| PASS | 기대대로 작동 |
| FAIL | 수정 필요 |
| N/A | 이번 납품 범위 아님 |

## 1. 정상 흐름 테스트

| ID | 테스트 | 입력 | 기대 결과 | 결과 |
|---|---|---|---|---|
| N-01 | 오늘 예약 생성 | 예약일이 기준일과 같은 예약 | 당일안내가 Send_Queue에 생성 | |
| N-02 | 내일 예약 생성 | 예약일이 기준일+1인 예약 | 전날안내가 Send_Queue에 생성 | |
| N-03 | 미방문 상태 | 상태를 미방문으로 변경 | 미방문안내가 Send_Queue에 생성 | |
| N-04 | 완료 상태 | 상태를 완료로 변경 | 리뷰요청이 Send_Queue에 생성 | |
| N-05 | 주간 리포트 | refreshWeeklyReport 실행 | 예약/완료/취소/미방문 수 집계 | |

## 2. 네트워크/서비스 장애 가정 테스트

| ID | 테스트 | 상황 | 기대 결과 | 결과 |
|---|---|---|---|---|
| O-01 | 폼 제출 불가 | 네트워크 끊김 | 수기 접수표로 대체 가능 | |
| O-02 | 시트 동기화 지연 | 직원이 오프라인 편집 | 복구 후 사전 점검 실행 | |
| O-03 | Apps Script 실행 실패 | 일시적 서비스 오류 | Audit_Log 또는 Validation_Errors 확인 후 재실행 | |
| O-04 | 자동 트리거 실패 | 오전 갱신 누락 | 수동으로 발송 대기함 갱신 가능 | |

## 3. 사람 실수 테스트

| ID | 테스트 | 입력 | 기대 결과 | 결과 |
|---|---|---|---|---|
| H-01 | 예약 ID 누락 | appointment_id 빈칸 | Validation_Errors에 기록 | |
| H-02 | 예약 ID 중복 | 같은 appointment_id 2개 | Validation_Errors에 중복 오류 기록 | |
| H-03 | 고객명 누락 | 고객명 빈칸 | Validation_Errors에 기록 | |
| H-04 | 연락처 누락 | 연락처 빈칸 | Validation_Errors에 기록 | |
| H-05 | 예약일 누락 | 예약일 빈칸 | Validation_Errors에 기록 | |
| H-06 | 상태값 오타 | `예약` 입력 | Validation_Errors에 기록 | |
| H-07 | 수신동의 오타 | `예` 입력 | Validation_Errors에 기록 | |
| H-08 | 템플릿 삭제 | 전날안내 템플릿 삭제 | 메시지본문에 템플릿 없음 표시 | |
| H-09 | 발송완료 체크 누락 | 대기 상태 유지 | Dashboard의 발송 대기 수에 남음 | |

## 4. 중복/동시 실행 테스트

| ID | 테스트 | 방법 | 기대 결과 | 결과 |
|---|---|---|---|---|
| D-01 | 발송 대기함 중복 방지 | refreshSendQueue 2회 실행 | 같은 예약/메시지/발송일 중복 생성 안 됨 | |
| D-02 | 동시 실행 잠금 | 두 사용자가 동시에 메뉴 실행 | 한 작업만 실행되고 충돌 방지 | |
| D-03 | queue_id 연속성 | 기존 행 삭제 후 새로 생성 | 기존 최대 번호 다음 값 생성 | |

## 5. 복구 테스트

| ID | 테스트 | 방법 | 기대 결과 | 결과 |
|---|---|---|---|---|
| R-01 | 백업 생성 | 백업 스냅샷 생성 실행 | 주요 시트 복사본 생성 | |
| R-02 | 잘못된 데이터 수정 | 오류 행 수정 후 사전 점검 | 오류 목록 감소 | |
| R-03 | 수식 삭제 복구 | 원본 템플릿에서 수식 복사 | 대시보드 정상 계산 | |
| R-04 | 대량 입력 전 백업 | 데이터 붙여넣기 전 백업 | 문제 시 백업 시트로 비교 가능 | |

## 6. 개인정보 안전 테스트

| ID | 테스트 | 확인 항목 | 기대 결과 | 결과 |
|---|---|---|---|---|
| P-01 | 민감정보 입력 금지 안내 | Form 요청사항 설명 | 진료/질병/검사결과 입력 금지 문구 존재 | |
| P-02 | 수신동의 필드 | Google Form | 수신 동의 질문 존재 | |

| P-04 | 공유 권한 | Google Sheet 공유 설정 | 필요한 사람만 접근 | |
| P-05 | 외부 작업자 권한 | 납품 후 권한 | 외부 작업자 권한 회수 | |

## 납품 승인 기준

아래 항목이 모두 PASS이면 고객에게 납품 가능합니다.

- 정상 흐름 테스트 5개
- 사람 실수 테스트 7개 이상
- 중복 방지 테스트 3개
- 백업 생성 테스트
- 개인정보 안전 테스트 5개
