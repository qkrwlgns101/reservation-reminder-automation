# 범용 예약·리마인더 자동화 패키지 MVP 사양서

## 1. 제품 개요

### 제품명
예약 누락 방지 자동화 패키지

### 한 줄 설명
예약 등록, 안내 문구 생성, 발송 대기함, 미방문 후속 관리, 주간 리포트를 한 번에 관리하는 소상공인용 예약·리마인더 자동화 시스템.

### 핵심 가치
- 직원이 매일 반복하는 예약 확인 연락 시간을 줄인다.
- 예약 누락, 당일 혼선, 노쇼를 줄인다.
- 고객 응대 문구를 표준화해 서비스 품질을 일정하게 유지한다.
- 업종별 템플릿만 바꾸면 병원, 학원, 미용실, 상담센터, PT샵, 사진관 등에 재판매할 수 있다.

### 1차 MVP 구현 방식
- Google Sheets + Google Forms + Apps Script
- 실제 발송은 직원이 확인 후 복사/발송
- 문자, 카카오 알림톡, 이메일 API 자동 발송은 2차 옵션

## 2. 타깃 고객

### 1차 타깃
- 의원, 치과, 피부과, 한의원
- 학원, 과외/상담 센터
- 미용실, 네일샵, 피부관리샵
- PT샵, 필라테스, 예약제 스튜디오
- 사진관, 정비소, 컨설팅 사무실

### 구매 의사결정자
- 원장, 대표, 실장, 매니저

### 주요 사용자
- 예약 담당 직원
- 상담 담당 직원
- 관리자/대표

## 3. 해결할 문제

| 문제 | 현재 방식 | MVP 해결 방식 |
|---|---|---|
| 예약 확인 연락 반복 | 직원이 매번 직접 작성 | 예약 정보 기반 문구 자동 생성 |
| 전날/당일 안내 누락 | 수기 확인 | 오늘 보낼 메시지 목록 자동 표시 |
| 예약 상태 관리 혼란 | 메모장, 카톡, 엑셀 분산 | 상태값으로 통합 관리 |
| 취소/미방문 후속 연락 누락 | 직원 기억에 의존 | 미방문 고객 재예약 문구 생성 |
| 운영 현황 파악 어려움 | 매번 수기 집계 | 주간 예약 리포트 자동 계산 |

## 4. MVP 범위

### 포함 기능
1. 예약 등록
2. 고객 정보 관리
3. 업종별 메시지 템플릿 관리
4. 예약 확정 문구 생성
5. 전날 리마인더 문구 생성
6. 당일 리마인더 문구 생성
7. 발송 대기함
8. 발송 완료 체크
9. 예약 상태 관리
10. 취소/미방문 고객 후속 문구 생성
11. 주간 리포트
12. 기본 설정 관리

### 제외 기능
- 진료, 법률, 세무 등 전문 판단 자동화
- 고객에게 AI가 직접 상담하는 챗봇
- EMR, POS, CRM 직접 연동
- 문자/카카오 알림톡 완전 자동 발송
- 결제 시스템
- 고객 본인 인증
- 대규모 권한 관리

## 5. 기본 업무 흐름

### 예약 등록 흐름
1. 직원이 예약 입력 폼 또는 예약 시트에 고객 정보를 입력한다.
2. 시스템이 예약 ID를 생성한다.
3. 예약 상태는 `예약됨`으로 설정된다.
4. 예약 확정 메시지가 자동 생성된다.
5. 발송 대기함에 예약 확정 메시지가 표시된다.

### 리마인더 흐름
1. 매일 오전, 시스템이 오늘 발송할 메시지를 계산한다.
2. 예약일 기준 전날인 고객에게 `전날 안내` 문구를 생성한다.
3. 예약 당일인 고객에게 `당일 안내` 문구를 생성한다.
4. 직원은 발송 대기함에서 문구를 확인한다.
5. 직원이 실제 문자/카톡으로 발송한 뒤 `발송완료`로 체크한다.

### 방문 후 상태 관리 흐름
1. 직원이 예약 상태를 `완료`, `취소`, `미방문` 중 하나로 변경한다.
2. `미방문`이면 재예약 안내 문구가 자동 생성된다.
3. `완료`이면 리뷰 요청 문구 또는 재방문 안내 문구를 생성할 수 있다.

### 주간 리포트 흐름
1. 주간 예약 수, 완료 수, 취소 수, 미방문 수를 자동 계산한다.
2. 미방문율, 완료율, 취소율을 계산한다.
3. 다음 주 예약 수와 발송 예정 메시지 수를 보여준다.
4. 관리자용 요약 문구를 생성한다.

## 6. 상태값 정의

| 상태 | 의미 | 다음 상태 |
|---|---|---|
| 예약됨 | 예약이 등록됨 | 완료, 취소, 미방문, 변경 |
| 변경 | 날짜/시간이 변경됨 | 예약됨, 완료, 취소, 미방문 |
| 완료 | 고객이 방문 또는 이용 완료 | 리뷰요청, 재방문안내 |
| 취소 | 고객이 사전 취소 | 재예약안내 |
| 미방문 | 예약 시간에 오지 않음 | 재예약안내 |
| 보류 | 확인 필요 | 예약됨, 취소 |

## 9. 기본 메시지 템플릿

### 예약 확정
```text
{고객명}님, {사업장명} 예약이 확정되었습니다.
일시: {예약일} {예약시간}
항목: {서비스명}
변경이 필요하시면 {전화번호}로 연락 부탁드립니다.
```

### 전날 안내
```text
{고객명}님, 내일 {예약시간} {사업장명} 예약 안내드립니다.
예약 항목: {서비스명}
방문이 어려우시면 미리 {전화번호}로 연락 부탁드립니다.
```

### 당일 안내
```text
{고객명}님, 오늘 {예약시간} 예약 안내드립니다.
장소: {주소}
예약 변경이 필요하시면 {전화번호}로 연락 부탁드립니다.
```

### 미방문 후속
```text
{고객명}님, 오늘 예약 시간에 방문이 확인되지 않아 연락드립니다.
다시 예약을 원하시면 {전화번호}로 연락 부탁드립니다.
```

### 리뷰 요청
```text
{고객명}님, 오늘 방문해주셔서 감사합니다.
이용 경험이 만족스러우셨다면 리뷰를 남겨주시면 큰 도움이 됩니다.
감사합니다. {사업장명}
```

## 10. 리마인더 생성 규칙

| 메시지 종류 | 생성 조건 | 기본 발송일 |
|---|---|---|
| 예약확정 | 신규 예약 등록 시 | 예약 등록일 |
| 전날안내 | 예약 상태가 예약됨이고 예약일 하루 전 | 예약일 - 1일 |
| 당일안내 | 예약 상태가 예약됨이고 예약 당일 | 예약일 |
| 미방문안내 | 상태가 미방문으로 변경됨 | 상태 변경일 |
| 리뷰요청 | 상태가 완료로 변경됨 | 완료 당일 또는 다음 날 |

## 12. 권한과 개인정보 원칙

### 기본 원칙
- 고객 연락처와 예약 안내에 필요한 최소 정보만 저장한다.
- 민감정보, 진료정보, 질병명, 검사결과, 상담 세부 내용은 저장하지 않는다.
- 메시지는 자동 발송하지 않고 직원 승인 후 발송하는 것을 1차 기본값으로 한다.
- 고객이 수신 동의하지 않은 경우 발송 대기함에서 경고 표시한다.

### 고객에게 안내할 문구 예시
```text
예약 안내 및 변경 안내를 위해 이름과 연락처를 수집합니다.
수집된 정보는 예약 관리 목적으로만 사용되며, 요청 시 삭제할 수 있습니다.
```

## 13. 납품 기준

### 필수 납품물
- 예약관리 Google Sheets 템플릿
- 예약 입력 Google Forms
- 업종별 메시지 템플릿 3종 이상
- 발송 대기함
- 주간 리포트
- 관리자용 사용 매뉴얼 1페이지
- 샘플 데이터 20건

### 완료 조건
- 예약을 입력하면 예약 목록에 정상 반영된다.
- 예약 확정 문구가 자동 생성된다.
- 예약 하루 전 고객이 발송 대기함에 표시된다.
- 예약 당일 고객이 발송 대기함에 표시된다.
- 상태를 미방문으로 바꾸면 후속 문구가 생성된다.
- 주간 리포트의 예약 수, 완료 수, 취소 수, 미방문 수가 계산된다.
