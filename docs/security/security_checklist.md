# 보안 납품 체크리스트

## 납품 전 필수 확인

| 항목 | 확인 |
|---|---|
| 고객 소유 Google 계정 또는 Workspace에 파일을 생성했다 | |
| 링크 공유가 꺼져 있다 | |
| 필요한 직원에게만 개별 공유했다 | |
| 외부 작업자 권한은 납품 후 회수하기로 합의했다 | |
| 관리자/주요 사용자 2단계 인증을 권장했다 | |
| Google Form에 개인정보 수집·이용 안내 문구를 넣었다 | |
| Google Form에 민감정보 입력 금지 문구를 넣었다 | |
| 요청사항 필드는 선택 항목으로 두었다 | |
| 수신 동의 질문을 필수로 설정했다 | |
| `사전 점검 실행` 결과 BLOCKER가 없다 | |
| `보안 점검 실행` 결과 공개 공유 위험이 없다 | |
| `백업 스냅샷 생성`을 실행했다 | |
| `보호 설정 적용`을 실행했다 | |
| `Masked_View` 생성 여부를 확인했다 | |
| 직원용 사용 안내를 전달했다 | |

## 정기 점검

| 주기 | 점검 항목 |
|---|---|
| 매일 | 발송 대기함, 보류 메시지, 오류 여부 |
| 매주 | 주간 리포트, 미방문율, Validation_Errors |
| 매월 | 공유 권한, 외부 사용자, 백업 시트, 보관기간 초과 데이터 |
| 담당자 변경 시 | 권한 회수/부여, 2단계 인증 확인 |
| 대량 수정 전 | 백업 스냅샷 생성 |

## 금지 사항

- 원본 시트를 링크가 있는 모든 사용자에게 공유하지 않는다.
- 원본 예약 데이터를 이메일 첨부파일로 보내지 않는다.
- 요청사항에 진료 내용, 질병명, 검사결과를 적지 않는다.
- 주민등록번호, 카드번호, 계좌번호를 수집하지 않는다.
- 외부 작업자 권한을 납품 후 방치하지 않는다.
- Apps Script 코드에 API 키나 비밀번호를 직접 적지 않는다.

## 고객에게 인수인계할 말

```text
이 파일은 고객 개인정보가 들어가는 원본 파일입니다.
링크 공유를 켜지 말고, 필요한 직원에게만 개별 공유하세요.
외부에 현황을 보여줘야 한다면 원본이 아니라 Masked_View를 사용하세요.
매월 한 번은 보안 점검과 백업 정리를 실행하는 것을 권장합니다.
```
