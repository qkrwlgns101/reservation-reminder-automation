# 보안·개인정보 보호 설계서

## 목적

예약·리마인더 자동화 패키지는 이름, 연락처, 예약일, 예약 항목 같은 개인정보를 다룹니다. 따라서 단순한 예약표가 아니라 개인정보 처리 시스템으로 보고 설계해야 합니다.

이 문서는 고객에게 제시할 보안 원칙, 운영 정책, 권한 관리, 보관기간, 사고 대응 기준을 정리한 자료입니다.

## 핵심 보안 원칙

1. 최소 수집: 예약 안내에 필요한 정보만 받는다.
2. 목적 제한: 예약 확인, 변경 안내, 리마인더, 미방문 후속 관리에만 사용한다.
3. 민감정보 배제: 진료 내용, 질병명, 검사결과, 주민등록번호, 결제정보는 받지 않는다.
4. 직원 승인: 고객에게 발송되는 문구는 직원 확인 후 발송한다.
5. 접근 제한: 필요한 직원만 Google Sheet에 접근한다.
6. 기록 유지: 자동화 실행, 오류, 발송 처리 이력을 로그로 남긴다.
7. 보관기간 설정: 예약 목적이 끝난 데이터는 정해진 기간 후 삭제 또는 마스킹한다.
8. 복구 가능성: 수정 전 백업 스냅샷을 만들고 문제 발생 시 복구한다.

## 권장 Google Workspace 보안 설정

| 영역 | 권장 설정 |
|---|---|
| 계정 | 관리자와 주요 사용자 2단계 인증 필수 |
| 비밀번호 | 개인 계정 재사용 금지, 고유 비밀번호 사용 |
| 파일 공유 | 링크가 있는 모든 사용자 접근 금지 |
| 기본 공유 | Restricted 또는 제한됨으로 시작 |
| 외부 공유 | 필요한 사람에게만 개별 초대 |
| 퇴사/담당 변경 | 즉시 접근권한 회수 |
| 관리자 계정 | 평상시 일반 계정 사용, 관리자 계정 상시 로그인 지양 |
| 백업 | 대량 수정 전 백업 스냅샷 생성 |
| 외부 작업자 | 구축 기간 후 권한 회수 |

Google Workspace 공식 보안 체크리스트도 소규모 사업장에 2단계 인증, 링크 공유 제한, 외부 공유 경고를 권장합니다.

## 시트 권한 설계

| 사용자 | 권장 권한 | 설명 |
|---|---|---|
| 대표/원장 | 편집자 | 전체 확인 가능 |
| 실장/관리자 | 편집자 | 예약/상태/발송 관리 |
| 일반 직원 | 제한 편집 또는 보기 | 발송 대기함 중심 사용 |
| 외부 구축자 | 임시 편집자 | 납품 후 권한 회수 |
| 고객/일반 사용자 | 접근 불가 | 폼 제출만 허용 |

## 시트별 보안 기준

| 시트 | 민감도 | 보호 방식 |
|---|---|---|
| Settings | 중 | 수정 전 백업, 일부 값만 수정 |
| Customers | 높음 | 외부 공유 금지, 마스킹 뷰 사용 |
| Appointments | 높음 | 상태/메모 입력 주의, 민감정보 금지 |
| Message_Templates | 낮음 | 변수 삭제 주의 |
| Send_Queue | 높음 | 발송상태와 확인자 기록 |
| Send_Log | 높음 | 삭제 지양, 접근 제한 |
| Weekly_Report | 중 | 개인 식별값 없이 집계 중심 |
| Validation_Errors | 중 | 오류 수정 후 보관 |
| Audit_Log | 중 | 삭제 지양 |
| Masked_View | 낮음 | 외부 공유가 필요할 때만 사용 |

## 보관기간 정책

기본 권장값:

- 예약 운영 데이터: 1년
- 발송 로그: 1년
- 오류 로그: 6개월
- 백업 스냅샷: 3개월
- 미방문/취소 사유: 6개월 또는 고객 요청 시 삭제

고객 업종과 내부 정책에 따라 보관기간은 조정할 수 있습니다.

## 삭제·마스킹 기준

| 상황 | 조치 |
|---|---|
| 고객 삭제 요청 | 해당 고객 행 삭제 또는 이름/연락처 마스킹 |
| 보관기간 경과 | 이름/연락처 마스킹 후 집계값만 유지 |
| 민감정보 발견 | 즉시 삭제, Audit_Log 기록 |
| 잘못 입력된 연락처 | 수정 전후 이력 기록 |
| 외부 공유 필요 | 원본 대신 Masked_View 공유 |

## 보안 점검 항목

Apps Script의 `보안 점검 실행` 메뉴는 아래 항목을 확인합니다.

- 파일 공유 상태가 공개 또는 링크 공유인지 확인
- 민감정보 의심 키워드가 요청사항/메모에 있는지 확인
- 주민등록번호, 카드번호처럼 수집 금지 패턴이 있는지 확인
- 연락처 형식이 과도하게 자유롭게 입력되지 않았는지 확인
- 보관기간을 초과한 예약 데이터가 있는지 확인
- 외부 공유용 마스킹 뷰가 생성되어 있는지 확인

## 보안 사고 대응 절차

### 1단계: 중단

- 공유 링크를 즉시 제한으로 변경
- 외부 작업자 권한 회수
- 자동 발송 연동이 있다면 중지

### 2단계: 확인

- `Audit_Log`에서 최근 실행 기록 확인
- Google Drive 공유 대상 확인
- `Validation_Errors`와 `Security_Audit` 확인
- 유출 가능성이 있는 행과 필드 범위 확인

### 3단계: 조치

- 잘못 공유된 파일 권한 회수
- 민감정보 삭제 또는 마스킹
- 백업본에 동일 정보가 있으면 백업본도 삭제 또는 마스킹
- 고객 또는 내부 책임자에게 필요한 범위만 보고

### 4단계: 재발 방지

- 접근권한 재정리
- 2단계 인증 확인
- 폼 질문 문구 강화
- 직원 교육
- 보관기간 자동 점검 주기 설정

## 고객에게 설명할 보안 문구

```text
이 시스템은 예약 안내에 필요한 최소 정보만 다룹니다.
진료 내용, 질병명, 검사결과, 주민등록번호, 결제정보는 수집하지 않습니다.
원본 시트는 필요한 직원에게만 공유하고, 외부 공유가 필요한 경우에는 연락처가 가려진 마스킹 뷰를 사용합니다.
자동화 실행 기록과 오류 기록을 남겨 문제가 생겼을 때 확인할 수 있도록 설계했습니다.
```
