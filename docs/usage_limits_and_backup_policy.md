# 무료 사용 한도와 백업 정책

이 프로젝트는 각 사용자가 본인의 Google 계정에서 본인의 Google Sheets 파일을 복사해 사용하는 것을 기본 운영 방식으로 합니다.

운영자가 모든 고객 데이터를 한곳에 모으는 중앙 SaaS 방식이 아니기 때문에 초기 배포, 보안, 비용 부담이 낮습니다. 대신 Google Sheets와 Apps Script의 무료 사용 한도 안에서 동작하므로, 사용자는 데이터 규모와 자동화 실행 횟수를 관리해야 합니다.

## 권장 운영 방식

| 항목 | 권장 방식 |
|---|---|
| 파일 소유자 | 실제 사용하는 병원, 매장, 개인 사업자의 Google 계정 |
| 데이터 저장 위치 | 사용자 개인 Google Drive |
| 운영자 접근 | 설치/점검 기간에만 임시 접근 권한 부여 |
| 실사용 데이터 | 이름, 연락처, 예약일시, 예약 상태 정도로 최소화 |
| 민감정보 | 진단명, 검사결과, 주민등록번호, 카드번호, 계좌번호 저장 금지 |
| 백업 | 시트 내부 스냅샷 + 월별 별도 파일 아카이브 병행 |

## 무료판 권장 사용량

아래 기준은 안정적인 무료 운영을 위한 보수적인 권장치입니다. Google 정책과 계정 종류에 따라 실제 한도는 달라질 수 있습니다.

| 기준 | 무료판 권장치 |
|---|---:|
| 월 예약 건수 | 300건 이하 |
| 누적 예약 행 | 3,000행 이하 |
| 직원 동시 편집 | 1~3명 |
| 자동 갱신 실행 | 하루 1~3회 |
| 메시지 대기열 생성 | 수동 확인 후 발송 |
| 백업 스냅샷 | 대량 수정 전 또는 주 1회 |
| 월별 아카이브 파일 | 매월 1회 |

다음 상태가 보이면 유료 유지보수, Google Workspace, 별도 데이터베이스, 또는 전용 웹앱 구조를 검토합니다.

- 예약 행이 3,000행을 넘어서 시트가 느려진다.
- 하루에도 여러 명이 동시에 수정해서 충돌이 잦다.
- Apps Script 실행 시간이 자주 초과된다.
- 문자, 카카오, 이메일 발송을 자동 API로 붙이고 싶다.
- 여러 지점이나 여러 담당자의 권한을 나누어야 한다.
- 개인정보 접근 로그, 보관 기간, 삭제 요청 대응을 엄격히 관리해야 한다.

## 백업 종류

### 1. 백업 스냅샷

메뉴:

```text
예약 자동화 > 백업 스냅샷 생성
```

현재 스프레드시트 안에 주요 시트의 숨김 복사본을 만듭니다.

사용 시점:

- 대량 붙여넣기 전
- 수식이나 템플릿을 수정하기 전
- Apps Script 코드를 바꾸기 전
- 직원 교육 전후
- 오류가 발생했지만 원인을 아직 모를 때

장점:

- 빠르게 만들 수 있습니다.
- 같은 파일 안에서 비교와 복구가 쉽습니다.

주의:

- 같은 파일 안에 있으므로 원본 파일 자체가 삭제되면 함께 사라질 수 있습니다.
- 백업 시트에도 개인정보가 들어 있으므로 공유 권한을 똑같이 조심해야 합니다.

### 2. 월별 아카이브 파일

메뉴:

```text
예약 자동화 > 월별 아카이브 파일 생성
```

현재 스프레드시트 전체를 같은 Google Drive 폴더에 별도 파일로 복사합니다.

파일명 예시:

```text
예약 누락 방지 자동화_archive_2026-06
```

사용 시점:

- 매월 말 또는 매월 1일
- 월간 리포트 확정 후
- 고객/예약 데이터 정리 전
- 운영자가 유지보수 계약을 종료하기 전

장점:

- 원본 파일과 분리된 복사본이 생깁니다.
- 월별 보관, 다운로드, 삭제가 쉽습니다.
- 실수로 원본 시트를 망가뜨렸을 때 복구 기준점으로 쓸 수 있습니다.

주의:

- 아카이브 파일도 개인정보 파일입니다.
- 공유 링크를 공개로 열지 않습니다.
- 보관 기간이 지난 아카이브는 삭제하거나 개인정보를 마스킹합니다.

## 로컬 다운로드 권장 시점

사용자가 Google Drive 외부에도 보관해야 한다면 다음 시점에 `파일 > 다운로드`로 엑셀 파일을 내려받게 안내합니다.

- 첫 세팅 완료 직후
- 매월 말 정산 후
- 대량 수정 전
- Apps Script 코드 수정 전
- 외부 담당자에게 점검을 맡기기 전
- Google 계정 이전 또는 담당자 변경 전

로컬 다운로드 파일은 PC 바탕화면에 방치하지 말고 암호화된 저장소나 접근 제한이 있는 폴더에 보관합니다.

## 무료판 사용자 안내 문구

```text
이 도구는 무료 Google Sheets 기반 자동화입니다.
각 사용자의 Google 계정과 Google Drive에 데이터가 저장되며, 운영자는 고객 데이터를 중앙 서버에 보관하지 않습니다.

무료 Google 서비스 한도 안에서 동작하므로 예약 데이터가 많아지거나 자동화 실행이 잦아지면 속도 저하 또는 실행 제한이 생길 수 있습니다.
안정적인 사용을 위해 월 예약 300건 이하, 누적 예약 3,000행 이하, 직원 동시 편집 1~3명 수준을 권장합니다.

대량 수정 전에는 '백업 스냅샷 생성'을 실행하고, 매월 1회는 '월별 아카이브 파일 생성'으로 별도 백업 파일을 만들어 주세요.
```

## 수익화 연결 포인트

무료 도구 자체는 누구나 쓰게 하되, 운영 수익은 아래 서비스에서 만듭니다.

| 유료 서비스 | 판매 이유 |
|---|---|
| 초기 설치 대행 | 비개발자는 Apps Script 설치와 권한 설정에서 막힙니다. |
| 월 1회 백업/보안 점검 | 개인정보 파일이라 정기 점검 니즈가 있습니다. |
| 월별 리포트 정리 | 원장이 바로 볼 수 있는 요약 자료가 필요합니다. |
| 데이터 정리/마이그레이션 | 누적 행이 많아지면 정리 작업이 필요합니다. |
| 업종별 커스터마이징 | 병원, 치과, 미용실, 학원마다 문구와 상태값이 다릅니다. |
| 문자/카카오 연동 | 무료판 이후 가장 자연스러운 유료 확장입니다. |
